Canvas 动画之游走在蜜蜂格

有一条线，它的行走路线是一个等边六边形，它是怎么行走的呢？有一条线，他的路线可以自由选择，它是怎么走的呢？有一条线，它的颜色可以随机出现，它是怎么做到的呢？有一条线，它的行走轨迹是慢慢消失的，它是怎么做到的呢？有一条线，它的周围总是围绕着一些小的火花，它是怎么做到的呢？今天就分享给大家。

效果中的难点

1.线的路线可以慢慢的消失。

之所以用 Canvas 来实现，还有一个原因，可以解决路线慢慢消失这个小小的但却有难度的效果。可以利用 Canvas 中的 globalCompositeOperation 属性来完美实现。在动画帧数不断变化的时候，加入如下代码，可以让黑色和新生成出来的方块同时显示在当前画布上，处于叠加的效果，从而实现路线慢慢消失的效果。

2.线的行走看起来像是在沿着蜂房的骨架边缘

不知道大家是否看过蜜蜂的蜂房是什么形状的。我从网上找了张蜂房的类似图片，希望能更好的展现线的行走路线。

从上图来看，每只小蜜蜂的房间特别像个等边六边形。那么我怎么才能让我的线按着等边六边形来走呢？其实还算很简单的。只要大家对儿时学过的三角函数就可以解决上面的问题。

首先定义一个时间变量( iTime )，用于检测本次路线走的时间，这就是我们已知的一个常量，也是六边形每条边的边长。等边六边形每个角度都是固定的，那就是120度，用 Math.sin 和 Math.cos 来计算每个路线的 x 和 y 坐标即可。

3.线的路线随机可选

首先把每条线可能走的方向先在小本本上都画好。先从起点开始画，对于起点来说可以有六种可能，分别是右下 -> 左上、右下 <- 左上、右上 -> 左下、右上 <- 左下、右上 -> 左下和右上 <- 左下。把他们横纵的弧度都存起来。每种路线走完之后就有两种可能了，再把这两种可能都和每个路线存在一起。

4.线的颜色可以随机出来

颜色选用 hsl( 色调、饱和度、亮度 )。这样一来只需要把色调取个随机数即可实现颜色随机的效果。有关 hsl 取色请自行查看相关资料，在这里不做详细分享。

涉及对象的相关方法详解

本次效果采用 JavaScript 的面向对象的模式完成的。分为两个对象 Rect 和 Hive。下面介绍一些对象中比较值得一提的方法。其他方法有可能在其他有关动画效果中已经有过介绍，在这里就不一一介绍了。

Hive 对象中的 bgBlack 方法

这个方法是在页面一开始先绘制一下黑色背景。如果不绘制，线路慢慢消失的效果显示的并不是那么尽善尽美。

Hive 对象中的 drawframe 方法

这个方法主要的作用就是帧动画，每次绘制一次带有透明度的背景，从而实现线路慢慢消失的效果。其次就是在少于 总线数( aLineLenth )的时候添加一个新的线。这里的关键是 globalCompositeOperation 属性。如果是 source-over ，那么在目标图像上显示源图像。如果是 lighter ，那么显示源图像 + 目标图像。

Rect 对象中的 reset 方法

这个方法主要用于重置这个方块。让他的初始坐标（ iPointX, iPointY )恢复到页面中间，并且本次移动的时间( iTime )归零。每次增加的弧度( iRadianX, iRadianY )也归零，从新开始选择。累计时间( iCumulativeTime )也是归零。

Rect 对象中的 move 方法

首先要增加本次移动的时间( iTime )，和累计时间( iCumulativeTime )。如果本次移动的时间( iTime )到达了本次规定的时间( iBaseTime )，那么本次移动的时间( iTime )归零，更新增加的弧度( iRadianX, iRadianY )并且在下一个路线的索引( aNextIndex )中随机选择一个。

接下来就是利用 Math.sin 和 Math.cos 累加相应的坐标了。坐标更新好了，改变在初始化生成好的颜色，之后画上去。20%的几率上面可以画上小星星。

最后如果这条线存在的累计时间( iCumulativeTime )超过了规定的总时间( iAllTime )，那么就让它重置 ( reset )。
