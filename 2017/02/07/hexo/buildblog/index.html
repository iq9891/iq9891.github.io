Hexo 搭建 Github 静态博客

看着网上各种博客，各种技术分享层出不穷，我也忍不住想要分享一下做前端这些年我所走的路，踩的坑，以及未来的一些计划。2017年2月7日，我开始持续更新此博客，更新频率一周一篇，更新内容不限，但都是本人原创。

静态博客怎么做呢？我首先想到的是 Wordpress 。但转念一想，我既然是做前端的，不如看看有关前端的静态博客有没有现成的 SSG 。经过了一些 Google ，最终还是选择了 Hexo 。接下来我就介绍一下我选择 Hexo 的原因。

hexo的优点

超快速度

Node.js 所带来的超快生成速度，让上百个页面在几秒内瞬间完成渲染。几乎如同步渲染，这边编辑完md文件，切换过去，刷个新就能看到结果。

支持 Markdown

Hexo 支持 GitHub Flavored Markdown 的所有功能，甚至可以整合 Octopress 的大多数插件。有了 Markdown 的支持，我可以不担心代码显示是否好看。而且用起来方便，就像写 READMD.md 一样。

一键部署

只需一条指令即可部署到 GitHub Pages, Heroku 或其他网站。

轻松上手

并不像 Wordpress 那样，还需要数据库，甚至还需要 Docker。简单易学，容易上手，只需懂得一点点 Node，一点点 Npm，一点点 Markdown 语法，就可以轻松无压力的上手。

丰富的插件

Hexo 拥有强大的插件系统，安装插件可以让 Hexo 支持 Jade, CoffeeScript。

安装相应的工具

1.安装 Node，如果有 Node 就此略过

2.安装 Hexo

3.创建项目

新建完成后，指定文件夹的目录如下：

_config.yml

网站的 配置 信息，您可以在此配置大部分的参数。

package.json

应用程序的信息。EJS, Stylus 和 Markdown renderer 已默认安装，您可以自由移除。

scaffolds

模版 文件夹。当您新建文章时， Hexo 会根据 scaffold 来建立文件。

Hexo的模板是指在新建的markdown文件中默认填充的内容。例如，如果您修改 scaffold/post.md 中的 Front-matter 内容，那么每次新建一篇文章时都会包含这个修改。

source

资源文件夹是存放用户资源的地方。除 posts 文件夹之外，开头命名为 ( 下划线 )的文件 / 文件夹和隐藏的文件将会被忽略。 Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes

主题( themes )文件夹。默认是 Landscape 。 Hexo 会根据主题来生成静态页面。

安装主题

或者直接到 releases 下载最新源码文件，重命名为 Cafe 放到博客 themes 目录下。

Cafe 需要 Hexo 2.4 及以上版本支持.

使用主题

修改博客配置文件 _config.yml 主题属性 theme 为 cafe 。

主题配置

主题 themes/cafe/_config.yml 文件内容参考说明配置

配置教程具体见 _config.yml 文件注释说明。
