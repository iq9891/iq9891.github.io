本效果思量再三，还是决定用 Canvas 来实现。所以在 Html 文件中必须有一个 canvas 标签。因为效果是充满整个浏览器的，所以需要把默认样式清一下，也就写在 Css 中了。 Html 和 Css 中的东西就不在这里讲了，有兴趣不明白的大家可以自己 Google 一下。本文主要讲一下 JavaScript 中的几个方法，及波浪的原理。

Water 对象中的 initVar 方法

这个方法是声明一些所需的变量。之所以这么写，是因为让整个 JavaScript 的结构看起来更加清晰。

Water 对象中的 initVertex 方法

这个方法的作用是生成所有顶点，其中不仅有顶点的坐标( x | y )、速度( vy )和差分值( diffPt )。在绘制顶点的时候，根据顶点的坐标( x | y )来确定顶点的位置。每次波纹震荡，起震点( iStartingPoint )的差分值( diffPt )都会变化。除起震点( iStartingPoint )的其他点根据差分值( diffPt )来判断位置。速度( vy )用于计算顶点的纵坐标( y )。

Water 对象中的 setCanvas 方法

这个方法是让整个 canvas 充满整个浏览器窗口。

Water 对象中的 drawWater 方法

这个方法主要是绘制水平面的方法，值得提一句的是在这里绘画的顶点是循环之前生成的 this.aVertexes 数组中的数据而绘制的。绘制的原理是先绘制顶点到浏览器窗口底部的线，然后绘制右边最后一点到整个浏览器窗口最底部的线，最后画个从浏览器右下角到左下角的线。线和顶点画好之后再填充颜色。

Water 对象中的 update 方法

这个方法是更新顶点位置的方法。通过震荡点( this.iStartingPoint )的差分值( diffPt )不断变化不断减小，并且循环顶点数组( this.aVertexes )。

Water 对象中的 updatePostion 方法

这个方法是更新所有顶点的位置。

更新差分值：起震点开始左边顶点的差分值是当前的差分值 - 两个顶点差分值的差 * 比例。索引不得大于缓冲( this.iBuffer )。起震点开始右边顶点的差分值等于起震点开始左边顶点的差分值。

更新速度：首先当前速度 + 差分值 + 浏览器窗口的二分之一 - 当前纵坐标( y )。更新之后再乘以减速的比例( this.iDeceleration )。之所以乘以减速的比例( this.iDeceleration )，是因为达到减速的目的，最后停下来。

Water 对象中的 drawframe 方法

这个方法是执行动画。首先先清空给 canvas 内的所有像素。其次更新顶点坐标，然后绘制水平面。最后根据起震点( this.iStartingPoint )的速度( vy )来判断是否继续动画。
