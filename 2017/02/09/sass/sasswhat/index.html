Sass

Sass 是什么

Sass 是什么呢？引用 Sass 官网的一句话： Sass is the most mature, stable, and powerful professional grade CSS extension language in the world. 这句话什么意思呢？ Sass 究竟是什么呢？ Sass 是世界上最成熟，稳定，功能强大的专业级 CSS 扩展语言。

在我看来， Sass 是一种基于 Ruby 编写的 CSS 预处理器，诞生于2007年，是最早也是最成熟的一款 CSS 预处理器语言，它可以使用变量、嵌套、混入、继承，运算，函数等功能，使得 CSS 的开发，变得简单清晰可维护，同时也大大节省了设计者的时间，提高了效率。 Sass 最后还是会编译出合法的 CSS 让浏览器使用，也就是说它本身的语法并不太容易让浏览器识别，因为它不是标准的 CSS 格式，在它的语法内部可以使用动态变量等，所以它更像一种极简单的动态语言。

Sass的语法

第一种被称为 Scss (Sassy CSS)，是一个 CSS 语法的扩展，文件的扩展名为 .scss 。从简单的英文单词的理解，可以理解为 Sass 的 CSS ，意思就是像书写 Sass 一样书写 CSS 。另外， Scss 理解大多数 CSS hacks 以及浏览器专属语法，例如IE 古老的 filter 语法。 Scss 还支持一些类似书写 JavaScript 等脚本语言独有的循环( each )等语法。

第二种就是 Sass 本身的语法，文件的扩展名为 .sass 。比较老的语法成为缩排语法（或者就称为 “Sass” ）， 提供了一种更简洁的 CSS 书写方式。 这种书写形式和 Stylus 的书写方式有些类似。它不使用花括号，而是通过缩排的方式来表达选择符的嵌套层级，而且也不使用分号，而是用换行符来分隔属性。 很多人认为这种格式比 Scss 更容易阅读，书写也更快速。 缩排语法具有 Sass 的所有特色功能， 虽然有些语法上稍有差异。

Sass的特色

1.完全兼容 CSS

Sass 不仅支持 CSS2 ，还支持 CSS3 。任何 CSS 的库，CSS 插件都可以无缝对接。

2.功能丰富

Sass 拥有比任何其他 CSS 扩展语言更多的功能和能力。让我在用 Sass 编写样式，就像飞一样。 Sass 核心团队不断工作，不仅跟上，而且保持领先。

3.绝对首选

一次又一次地，业界选择 Sass 作为首要的 CSS 扩展语言。

4.强大的社区

Sass 由几个科技公司和数百个开发人员的联盟积极支持和开发。有了强大的社区，我们在开发之中遇到了任何问题都可以得到很好的解决。让我们在开发的过程中不在孤独。

5.轻松编译

选择Sass，我们不用担心怎么编译。因为有很多能让Sass快速编译成CSS的工具，如 Compass, Bourbon and Susy。

总结

Sass 就像是一种预处理器，是 CSS 的扩展。其特点有完全兼容 CSS 、功能丰富、绝对首选、强大的社区、轻松编译。用 Sass 编写 CSS 可以实现一些 CSS 不能实现的功能，比如模块化。

通过以上介绍，相比大家已经知道了 Sass 是什么了吧。如果你对 Sass 就此产生了浓厚的兴趣，那么还请翻看本博客其他关于 Sass 的文章吧。
